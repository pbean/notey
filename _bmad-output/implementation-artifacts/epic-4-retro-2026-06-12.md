## Successes

1. **Fourth consecutive 100%-completion epic, zero incidents, zero hard blockers.** Consistent delivery across Epics 1–4.

2. **The hardest story landed clean because of prep.** Story 4.4 (multi-instance CodeMirror) started with the answer already in hand thanks to the `codemirror-multi-instance-research.md` doc — itself an Epic 3 retro action item. Prep-de-risks-complexity, proven a second time.

3. **The `switchIdRef` cancellation pattern** (abort stale async after every `await`) emerged as the reusable architectural backbone for multi-tab editing — explicitly marked KEEP. Replaced an earlier bail-and-drop approach that silently lost tab switches.

4. **Debt was actively retired, not just logged.** Six inherited Epic 4 deferred items were closed: macOS Cmd-key parity, layout-mode CSS, the search↔palette↔note-list circular-import ring (via a clean `overlays/manager.ts` registry), `reorderTabs` integer validation, orphan-tab-on-load-failure, and symmetric overlay mutual exclusion.

5. **bmad-auto handoff was a non-event — in the best way.** Story 4.9 went spec → implement → review unattended and came out at the same quality bar. Pinkyd: "super happy with bmad-auto."

6. **Strong, reused patterns:** store-first/UI-later sequencing (4.1 built+tested `reorderTabs` long before its 4.7 UI trigger), reading event state from `dataTransfer` at event time (more robust than React state in handlers), shared-utility extraction (`formatRelativeDate`), and the restore-drives-existing-effect insight in 4.9 (bulk `restoreTabs` reused EditorPane's tab-switch effect — no new load trigger needed).

7. **Previous-retro follow-through: 5/5 Epic 3 action items done** — CodeMirror research doc, shared `resetAllStores()` test utility + DOM cleanup, ESLint unhandled-IPC rules (no-floating-promises + no-misused-promises), shadcn spike, trim-consistency fix. Continuity habit holding strong across four epics.

## Challenges

1. **Cleanup that closed 6 items quietly opened 2 new ones.** Both are real and still open:
   - **Persisted `layoutMode` (and theme) not applied at startup** — `main.tsx` applies `.dark` at boot but not `layoutMode`; restart silently ignores the saved layout preference. Latent UX bug.
   - **`toggleLayoutMode` / `toggleTheme` lack a concurrency guard** — two rapid invocations both read `getConfig` before either writes; one toggle is lost. We already solved this shape with `isCreatingNote` in 4.6 — just not applied here.

2. **Frontend test growth was minimal (~+7 across 9 stories).** Either multi-tab logic is genuinely hard to exercise in jsdom (records support this — no `DragEvent`/`clientX`, no real `ResizeObserver`, unsafe `navigator.platform`) or coverage leaned on review instead of tests. With bmad-auto driving, weak jsdom coverage matters *more* — less human eyeballing per run.

3. **E2E coverage still at zero — now four epics running.** Recurring deferred item from Epics 1, 2, 3. Consciously deferred again this retro, but the streak is explicitly flagged so it doesn't become permanent by default.

4. **jsdom remained the dominant test-quality bottleneck.** Forced a manual `dispatchDrag` MouseEvent helper (4.7), a `ResizeObserver` guard (4.2), and lazy `navigator.platform` evaluation (4.5).

5. **Reactive, story-by-story hardening on cross-cutting concerns.** Overlay mutual-exclusion and keyboard-shortcut suppression were patched incrementally (4.3 → 4.5 → 4.6 → 4.8) before being consolidated. The overlay-manager registry should have been the design from the start, not the cleanup.

---

## Key Insights

1. **Cleanup needs the same review rigor as features.** Closing 6 debt items introduced 2 new ones — both avoidable, both shapes we'd already solved elsewhere. A cleanup pass should re-check the patterns it touches against existing solutions (startup-config application, concurrency guards).

2. **bmad-auto raises the stakes on shifted-left quality.** Unattended implementation works well, but it amplifies the cost of weak automated coverage. The E2E gap and the thin frontend-test growth are now higher-priority than when humans reviewed every diff.

3. **Prep-de-risks-complexity, confirmed twice.** CodeMirror research → clean 4.4, exactly as FTS5 research → clean 3.1. Make this a standing pattern for the highest-risk story of every epic.

4. **Cross-cutting concerns deserve up-front design.** Overlay coordination was the single most-patched concern of the epic. When a concern touches 3+ stories, design the shared abstraction before the first consumer, not after the third.

5. **Bugs this epic were boundary/identity bugs, not logic bugs** — `splice(NaN)`→index 0 (4.7), stale `editorView`→wrong tab (4.9), out-of-bounds `selectedIndex` (4.8). Validation and staleness checks, not algorithm errors. Targeted property/boundary tests would catch these earlier than review.
