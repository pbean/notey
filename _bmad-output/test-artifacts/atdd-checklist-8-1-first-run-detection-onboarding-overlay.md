## Green-Phase Wiring (do this when implementing)

1. **Backend persistence:** implement `services/onboarding.rs` against `onboarding.toml` in `services::config::config_dir()` (atomic temp+rename, like `config::save`). NOTE: AC says "in config" — this scaffold uses a sibling `onboarding.toml` to avoid changing `AppConfig` (which would ripple to ~50 TS literal sites). Decide at green phase whether to fold the flag into `[general]` instead.
2. **Commands + bindings:** add `get_onboarding_state` / `complete_onboarding` (and optionally `increment_onboarding_session`) Tauri commands; register in `lib.rs::specta_builder`; add permission TOMLs under `src-tauri/permissions/autogenerated/`; add identifiers to `capabilities/default.json` and `EXPECTED_COMMANDS` in `tests/acl_tests.rs` (known Tauri v2 gotcha — see project-context). Then rebuild to regenerate `src/generated/bindings.ts`.
3. **Frontend api:** wire `src/features/onboarding/api.ts` to the generated `commands.*` (NEVER raw `invoke`).
4. **Overlay component:** render the markup listed in the component-test assertions; wire Esc + hotkey-press to `dismiss()`; mount `<OnboardingOverlay />` in the app shell and call `init(globalShortcut)` at startup; increment the session counter on launch.
5. **Status bar:** show the "Ctrl+P for commands" hint gated by `useOnboardingStore.shouldShowCommandHint()`.
